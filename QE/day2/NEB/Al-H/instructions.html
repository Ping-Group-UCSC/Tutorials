A very brief reminder of useful linux commands:

  • cp A B copies file A to file B.

  • mv A B moves file A to file B.

  • mkdir C creates a directory named C.

  • ls C lists the names of the files and sub-directories contained in directory C

  • vi A or gedit A are some of the many ways in which you can open the file A for reading and/or editing.

  • Adding an ampersand & at the end of your command will make it run in the background, so that you can continue to issue other commands while the original one is still running.


In this exercise, you will compute the reaction of a hydrogen molecule transferring across an aluminum surface.


    Realistically a slab of Al should be used, but to reduce the computational cost here we only consider a single layer of Al.

    For full details of the neb.x input refer to the link below:

    http://web.mit.edu/espresso_v6.1/i386_linux26/qe-6.1/Doc/INPUT_NEB.html


    • Run the NEB calculation by executing:


      neb.x -i neb.in > neb.out


    • This calculation consists of several geometry optimization calculations and may take a while (~5 minutes).

    • So feel free to take a break or work on something else while it runs.

    • After it is done, you can look through the output (neb.out) to see the convergence of optimizing 3 intermediate images which yield a barrier from image 1 to image 5

    • Look at the activation energy to see the amount of energy it takes to undergo the reaction


      grep "\->" neb.out


    • Be careful to type the above line correctly! You may accidentally write over your output file.

    • How does the activation energy compare with kT at room temperature?

    • Use xcrysden to visualize the transfer of H across the Al surface.


      xcrysden --axsf Al-H.axsf


    • The data file Al-H.int contains the reaction barrier data. Use gnuplot or xmgrace to plot this file and see the reaction barrier as a function of reaction coordinate.
